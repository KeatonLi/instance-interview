# 简历优化功能

## 一、功能概述

AI 智能优化简历措辞，帮助用户提升简历质量，增加面试通过率。

## 二、用户场景

用户在简历编辑器中，希望优化某段工作经历或项目描述的表述。

## 三、功能流程

```
1. 用户选中要优化的内容（工作经历 / 项目描述）
2. 点击「AI 优化」按钮
3. 弹出优化对话框，显示原文
4. AI 返回优化后的内容
5. 用户预览优化结果
6. 确认应用 or 取消
```

## 四、UI 设计

在简历编辑器的"工作经历"和"项目经验"区块，每个条目添加"优化"按钮。

## 五、API 设计

### 5.1 优化简历内容

**接口：** `POST /api/v1/resumes/optimize`

**认证：** 需要

**请求参数：**
```json
{
  "content": "原简历内容文本",
  "type": "work_experience | project | education | award",
  "optimize_type": "措辞优化 | 关键词增强 | 成就量化 | 全部"
}
```

**响应：**
```json
{
  "code": 0,
  "message": "优化成功",
  "data": {
    "original": "原内容",
    "optimized": "优化后内容",
    "changes": ["使用更专业的动词", "量化成果"]
  }
}
```

## 六、技术实现

### 后端

1. 调用 MiniMax API
2. 构建 Prompt（包含优化指令和上下文）
3. 解析返回结果
4. 返回优化内容

### 前端

1. 在 ResumeForm 中添加"优化"按钮
2. 调用优化 API
3. 显示优化结果
4. 用户确认后更新内容

## 七、优化类型说明

| 类型 | 说明 | Prompt 关键词 |
|------|------|--------------|
| 措辞优化 | 使描述更专业、简洁 | "professional", "concise" |
| 关键词增强 | 增强 ATS 关键词 | "ATS keywords", "recruiter" |
| 成就量化 | 量化工作成果 | "quantify", "metrics", "%" |
| 全部 | 综合优化 | "comprehensive" |

## 八、注意事项

1. 保留原文，让用户对比
2. 优化失败时显示友好提示
3. 限制单次优化内容长度（最多 500 字）
4. 考虑添加"历史优化"功能（可选）
